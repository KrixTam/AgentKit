## 一、Skill 与 Agent 的边界 — 维持清晰分离

### 决策

保持 **Skill 是"指令 + 资源包"、Agent 是"LLM 配置单元"** 的清晰边界。

### 边界规则

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  Agent                          Skill                            │
│  ─────                          ─────                            │
│  ✅ 有自己的 LLM 配置            ✅ 有指令和资源                   │
│  ✅ 有自己的对话循环              ❌ 没有自己的对话循环              │
│  ✅ 可以持有工具和 Skill          ✅ 可以声明需要的额外工具          │
│  ✅ 可以交接/被交接               ❌ 不能独立被交接                  │
│  ✅ 有回调和护栏                  ❌ 不能有自己的回调                │
│  ✅ 有状态（session state）       ❌ 无独立状态（借用 Agent 的）     │
│                                                                  │
│  关系：Agent 通过 SkillToolset 使用 Skill                        │
│  类比：Agent = 工程师，Skill = 操作手册                           │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 如果需要"更智能的 Skill"怎么办？

当一个 Skill 需要更复杂的逻辑时，正确的做法是：

```python
# ❌ 错误：让 Skill 自带 Agent
skill = Skill(
    name="complex-skill",
    sub_agent=Agent(...)  # 不要这样做
)

# ✅ 正确：创建专门的 Agent，把 Skill 赋给它
complex_agent = Agent(
    name="complex_task_handler",
    instructions="...",
    skills=[complex_skill],
    model="gpt-4o",
)

# 然后让主 Agent 通过 handoff 或 as_tool 使用它
main_agent = Agent(
    name="coordinator",
    tools=[complex_agent.as_tool("do_complex_task", "处理复杂任务")],
)
```

## 二、Skill 的版本管理与分发 — 包管理器方案

### 决策

采用**方案 B**：类似 npm 的 Skill 包管理器。

### SkillPackageManager 设计

#### 用户体验（CLI）

```bash
# 安装 Skill
skill install pdf-processor
skill install pdf-processor@1.2.0          # 指定版本
skill install ./local-skills/my-skill      # 从本地安装

# 管理 Skill
skill list                                  # 列出已安装的 Skill
skill info pdf-processor                    # 查看详情
skill update pdf-processor                  # 更新到最新版
skill uninstall pdf-processor               # 卸载

# 发布 Skill
skill publish                               # 发布当前目录的 Skill
skill validate                              # 校验 SKILL.md 格式
```

#### 配置文件：`skill.lock`

类似 `package-lock.json`，记录项目中所有 Skill 的精确版本：

```yaml

skills:
  pdf-processor:
    version: "1.2.0"
    integrity: "sha256-abc123..."
    source: "registry"
  data-analysis:
    version: "0.3.1"
    integrity: "sha256-def456..."
    source: "registry"
  my-custom-skill:
    version: "local"
    path: "./skills/my-custom-skill"
    source: "local"
```

#### 项目配置：`skill.toml`

```toml
[project]
name = "my-agent-project"
version = "1.0.0"

[skills]
pdf-processor = "^1.2.0"
data-analysis = ">=0.3.0"

[skills.sources]

[skills.install_path]
```

#### 核心类设计

```python
class SkillPackageManager:
    """Skill 包管理器"""
    
    def __init__(self, project_dir: Path, registry_url: str = DEFAULT_REGISTRY):
        self.project_dir = project_dir
        self.registry = SkillRegistry(registry_url)
        self.install_dir = project_dir / ".skills"
        self.lockfile = project_dir / "skill.lock"
    
    async def install(self, name: str, version: str = "latest") -> Skill:
        """安装 Skill"""
        # 1. 从 registry 查询可用版本
        package_info = await self.registry.resolve(name, version)
        
        # 2. 下载并解压
        archive = await self.registry.download(package_info)
        skill_dir = self._extract(archive, self.install_dir / name)
        
        # 3. 安全检查（前置审查）
        audit_result = await SkillAuditor.audit(skill_dir)
        if audit_result.has_critical_issues:
            raise SecurityError(f"Skill '{name}' 存在安全风险: {audit_result.summary}")
        
        # 4. 加载并验证
        skill = load_skill_from_dir(skill_dir)
        
        # 5. 更新 lockfile
        self._update_lockfile(name, package_info)
        
        return skill
    
    async def publish(self, skill_dir: Path) -> str:
        """发布 Skill 到 registry"""
        # 1. 格式校验
        validate_skill_structure(skill_dir)
        
        # 2. 打包
        archive = self._package(skill_dir)
        
        # 3. 上传
        url = await self.registry.publish(archive)
        return url

class SkillRegistryServer:
    """Skill 注册中心（服务端）"""
    
    # 存储结构类似 npm registry：
    # /{skill-name}/
    #   ├── metadata.json       # 版本列表、描述、作者
    #   └── versions/
    #       ├── 1.0.0.tar.gz
    #       ├── 1.1.0.tar.gz
    #       └── 1.2.0.tar.gz
```

## 三、Skill 的安全性 — 沙箱执行方案

### 决策

采用**方案 A**：沙箱执行，**三级防护体系**。

### 三级沙箱防护

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  Level 1：临时目录隔离（默认，轻量）                              │
│  ─────────────────────────────────                                │
│  • 所有 Skill 资源解压到 tempdir                                  │
│  • 脚本在 tempdir 中执行，无法访问宿主文件系统                    │
│  • 执行完毕后自动清理                                            │
│  • 适用于：受信任的 Skill（自己开发的 / 已审计的）                │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Level 2：子进程隔离（中等安全）                                  │
│  ─────────────────────────                                       │
│  • 在独立子进程中执行脚本                                        │
│  • 限制：超时控制、内存上限、无网络访问（可选）                   │
│  • 通过 stdout/stderr 收集结果                                   │
│  • 适用于：第三方 Skill（从 registry 安装的）                    │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Level 3：Docker 容器隔离（最高安全）                             │
│  ───────────────────────────────                                  │
│  • 在轻量 Docker 容器中执行                                      │
│  • 完全隔离的文件系统、网络、进程空间                             │
│  • 可配置资源限制（CPU、内存、磁盘）                              │
│  • 适用于：不受信任的 Skill / 生产环境                           │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### SandboxExecutor 设计

```python
from abc import ABC, abstractmethod
from enum import Enum

class SandboxLevel(Enum):
    TEMPDIR = "tempdir"         # Level 1: 临时目录
    SUBPROCESS = "subprocess"   # Level 2: 子进程隔离
    DOCKER = "docker"           # Level 3: Docker 容器

class SandboxConfig(BaseModel):
    """沙箱配置"""
    level: SandboxLevel = SandboxLevel.SUBPROCESS    # 默认 Level 2
    timeout_seconds: int = 60                         # 超时（秒）
    max_memory_mb: int = 512                          # 最大内存（MB）
    allow_network: bool = False                       # 是否允许网络
    allow_env_vars: list[str] = []                    # 允许的环境变量白名单
    docker_image: str = "python:3.12-slim"            # Docker 镜像（Level 3 用）

class BaseSandboxExecutor(ABC):
    """沙箱执行器基类"""
    
    @abstractmethod
    async def run(self, skill: Skill, script: SkillScript, 
                  arguments: dict, config: SandboxConfig) -> "ExecutionResult": ...

class ExecutionResult(BaseModel):
    """脚本执行结果"""
    stdout: str = ""
    stderr: str = ""
    exit_code: int = 0
    timed_out: bool = False
    error: Optional[str] = None
    
    @property
    def success(self) -> bool:
        return self.exit_code == 0 and not self.timed_out

# ===== Level 1: 临时目录隔离 =====

class TempDirExecutor(BaseSandboxExecutor):
    """在临时目录中执行脚本"""
    
    async def run(self, skill, script, arguments, config):
        import tempfile, subprocess
        
        with tempfile.TemporaryDirectory() as tmpdir:
            # 1. 将 Skill 资源写入临时目录
            self._materialize_resources(skill, tmpdir)
            
            # 2. 在临时目录中执行
            result = subprocess.run(
                ["python", f"scripts/{script.filename}"] + self._args_to_list(arguments),
                cwd=tmpdir,
                capture_output=True,
                timeout=config.timeout_seconds,
                text=True,
            )
            
            return ExecutionResult(
                stdout=result.stdout,
                stderr=result.stderr,
                exit_code=result.returncode,
            )

# ===== Level 2: 子进程隔离 =====

class SubprocessExecutor(BaseSandboxExecutor):
    """在隔离子进程中执行脚本"""
    
    async def run(self, skill, script, arguments, config):
        import asyncio, tempfile, resource
        
        with tempfile.TemporaryDirectory() as tmpdir:
            self._materialize_resources(skill, tmpdir)
            
            # 构建隔离的环境
            env = {k: os.environ[k] for k in config.allow_env_vars if k in os.environ}
            env["HOME"] = tmpdir
            
            # 使用 asyncio 子进程，支持超时
            try:
                proc = await asyncio.create_subprocess_exec(
                    "python", f"scripts/{script.filename}",
                    *self._args_to_list(arguments),
                    cwd=tmpdir,
                    env=env,
                    stdout=asyncio.subprocess.PIPE,
                    stderr=asyncio.subprocess.PIPE,
                    # 设置资源限制
                    preexec_fn=lambda: resource.setrlimit(
                        resource.RLIMIT_AS, 
                        (config.max_memory_mb * 1024 * 1024, -1)
                    ),
                )
                stdout, stderr = await asyncio.wait_for(
                    proc.communicate(),
                    timeout=config.timeout_seconds,
                )
                return ExecutionResult(
                    stdout=stdout.decode(),
                    stderr=stderr.decode(),
                    exit_code=proc.returncode,
                )
            except asyncio.TimeoutError:
                proc.kill()
                return ExecutionResult(timed_out=True, error="执行超时")

# ===== Level 3: Docker 容器隔离 =====

class DockerExecutor(BaseSandboxExecutor):
    """在 Docker 容器中执行脚本"""
    
    async def run(self, skill, script, arguments, config):
        import docker
        
        client = docker.from_env()
        
        # 1. 准备挂载目录
        tmpdir = tempfile.mkdtemp()
        self._materialize_resources(skill, tmpdir)
        
        try:
            # 2. 运行容器
            container = client.containers.run(
                image=config.docker_image,
                command=["python", f"/workspace/scripts/{script.filename}"]
                        + self._args_to_list(arguments),
                volumes={tmpdir: {"bind": "/workspace", "mode": "ro"}},
                mem_limit=f"{config.max_memory_mb}m",
                network_disabled=not config.allow_network,
                detach=True,
                remove=False,
            )
            
            # 3. 等待完成（带超时）
            result = container.wait(timeout=config.timeout_seconds)
            logs = container.logs(stdout=True, stderr=True).decode()
            
            return ExecutionResult(
                stdout=logs,
                exit_code=result["StatusCode"],
            )
        finally:
            container.remove(force=True)
            shutil.rmtree(tmpdir)

# ===== 统一入口 =====

class SandboxExecutorFactory:
    """根据配置创建对应的沙箱执行器"""
    
    @staticmethod
    def create(config: SandboxConfig) -> BaseSandboxExecutor:
        match config.level:
            case SandboxLevel.TEMPDIR:
                return TempDirExecutor()
            case SandboxLevel.SUBPROCESS:
                return SubprocessExecutor()
            case SandboxLevel.DOCKER:
                return DockerExecutor()
```

### Skill 安全审计

安装 Skill 时的前置安全检查：

```python
class SkillAuditor:
    """Skill 安全审计器"""
    
    # 危险模式检测
    DANGEROUS_PATTERNS = [
        r"os\.system\(",
        r"subprocess\.(?:call|run|Popen)\(",
        r"eval\(",
        r"exec\(",
        r"__import__\(",
        r"shutil\.rmtree\(",
        r"open\(.*/etc/",
        r"requests\.(?:get|post)\(",  # 未声明的网络访问
    ]
    
    @classmethod
    async def audit(cls, skill_dir: Path) -> "AuditResult":
        """审计 Skill 目录"""
        issues = []
        
        # 1. 检查 SKILL.md 格式
        # 2. 扫描脚本中的危险模式
        for script_file in (skill_dir / "scripts").glob("*.py"):
            content = script_file.read_text()
            for pattern in cls.DANGEROUS_PATTERNS:
                if re.search(pattern, content):
                    issues.append(AuditIssue(
                        severity="warning",
                        file=script_file.name,
                        description=f"检测到潜在危险操作: {pattern}",
                    ))
        
        # 3. 检查文件大小
        total_size = sum(f.stat().st_size for f in skill_dir.rglob("*") if f.is_file())
        if total_size > 50 * 1024 * 1024:  # 50MB
            issues.append(AuditIssue(severity="info", description="Skill 体积较大"))
        
        return AuditResult(issues=issues)
```

## 四、多模型支持策略 — 完全自研 LLM 适配层（重点）

### 决策变更

经评审决定**放弃 LiteLLM**，完全自研 LLM 适配层。

### 设计方案

详细设计已独立成文档：**[LLM 统一适配层 — 自研方案详细设计](./LLM统一适配层_自研方案设计.md)**

### 核心要点

**1. 只需 4 个适配器覆盖所有主流 LLM：**

| 适配器 | 覆盖厂商 | 说明 |
|--------|---------|------|
| `OpenAIAdapter` | OpenAI GPT 系列 | 原生 `openai` SDK |
| `AnthropicAdapter` | Anthropic Claude 系列 | 原生 `anthropic` SDK，差异最大 |
| `GoogleAdapter` | Google Gemini 系列 | 原生 `google-genai` SDK |
| `OpenAICompatibleAdapter` | 通义千问/智谱/DeepSeek/Moonshot/百川/Azure | 复用 `openai` SDK，仅改 api_base |

**2. 统一类型系统**：框架内部使用统一的 `Message` / `ToolDefinition` / `ToolCall` / `LLMResponse` 类型，各适配器负责与厂商格式的双向转换。

**3. 模型自动路由**：`LLMRegistry` 根据模型标识前缀（如 `gpt-` / `claude-` / `deepseek/`）自动选择适配器，零配置即可使用。

**4. Skill 级 LLM 切换**：Skill 在 SKILL.md 的 frontmatter 中声明 `llm_config`，激活时临时切换 LLM。

**5. 生产级特性**：内置重试（指数退避）、模型降级、成本追踪。

## 五、状态持久化 — Mem0 集成方案

### 决策

采用**方案 C**：集成 Mem0 作为默认记忆系统，同时保留抽象接口以支持替换。

### Mem0 核心能力（调研结果）

| 能力 | 说明 |
|------|------|
| **自动记忆提取** | 从对话中自动识别和提取值得记住的信息 |
| **长期记忆** | 跨会话保持用户偏好、历史决策、知识积累 |
| **向量存储** | 基于向量数据库做语义检索（支持 Qdrant/ChromaDB/Pinecone） |
| **图记忆** | 可选的知识图谱存储，建立实体关系 |
| **多用户隔离** | 支持 user_id / agent_id / session_id 三层隔离 |
| **Python SDK** | `pip install mem0ai`，API 简洁 |
| **自托管/云端** | 支持本地部署和 Mem0 Cloud |

### 架构设计

```python
from abc import ABC, abstractmethod
from typing import Optional

# ===== 记忆抽象层 =====

@dataclass
class Memory:
    """一条记忆"""
    id: str
    content: str                    # 记忆内容
    metadata: dict = field(default_factory=dict)
    created_at: Optional[str] = None
    updated_at: Optional[str] = None
    score: float = 0.0             # 相关性分数（检索时）

class BaseMemoryProvider(ABC):
    """记忆提供者抽象接口"""
    
    @abstractmethod
    async def add(self, content: str, *, user_id: str = None, 
                  agent_id: str = None, metadata: dict = None) -> list[Memory]:
        """添加记忆（Mem0 会自动从内容中提取关键信息）"""
        ...
    
    @abstractmethod
    async def search(self, query: str, *, user_id: str = None,
                     agent_id: str = None, limit: int = 10) -> list[Memory]:
        """语义搜索相关记忆"""
        ...
    
    @abstractmethod
    async def get_all(self, *, user_id: str = None, 
                      agent_id: str = None) -> list[Memory]:
        """获取所有记忆"""
        ...
    
    @abstractmethod
    async def delete(self, memory_id: str) -> bool:
        """删除记忆"""
        ...

# ===== Mem0 实现 =====

class Mem0Provider(BaseMemoryProvider):
    """基于 Mem0 的记忆提供者"""
    
    def __init__(self, config: dict = None):
        from mem0 import Memory as Mem0Memory
        self._mem0 = Mem0Memory.from_config(config or self._default_config())
    
    async def add(self, content, *, user_id=None, agent_id=None, metadata=None):
        messages = [{"role": "user", "content": content}]
        result = self._mem0.add(
            messages,
            user_id=user_id,
            agent_id=agent_id,
            metadata=metadata or {},
        )
        return [Memory(id=r["id"], content=r["memory"]) for r in result.get("results", [])]
    
    async def search(self, query, *, user_id=None, agent_id=None, limit=10):
        results = self._mem0.search(
            query,
            user_id=user_id,
            agent_id=agent_id,
            limit=limit,
        )
        return [
            Memory(id=r["id"], content=r["memory"], score=r.get("score", 0))
            for r in results.get("results", [])
        ]
    
    async def get_all(self, *, user_id=None, agent_id=None):
        results = self._mem0.get_all(user_id=user_id, agent_id=agent_id)
        return [Memory(id=r["id"], content=r["memory"]) for r in results.get("results", [])]
    
    async def delete(self, memory_id):
        self._mem0.delete(memory_id)
        return True
    
    @staticmethod
    def _default_config():
        return {
            "vector_store": {
                "provider": "qdrant",
                "config": {"collection_name": "agent_memories", "host": "localhost", "port": 6333}
            },
        }

class Agent(BaseAgent):
    """更新后的 Agent — 支持记忆"""
    
    memory: Optional[BaseMemoryProvider] = None
    
    async def _run_impl(self, ctx):
        instructions = await self.get_instructions(ctx)
        
        # ⭐ 自动检索相关记忆，注入到 LLM 上下文
        if self.memory:
            relevant_memories = await self.memory.search(
                ctx.input,
                user_id=ctx.user_id,
                agent_id=self.name,
                limit=5,
            )
            if relevant_memories:
                memory_text = "\n".join([f"- {m.content}" for m in relevant_memories])
                instructions += f"\n\n## 相关记忆\n{memory_text}"
        
        # ... 正常的 LLM 调用流程 ...
        
        # ⭐ 对话结束后，自动提取并存储新记忆
        if self.memory and final_output:
            conversation = f"User: {ctx.input}\nAssistant: {final_output}"
            await self.memory.add(
                conversation,
                user_id=ctx.user_id,
                agent_id=self.name,
            )
```
